# Mars Mission Log Analysis Report

## 1. 개요

화성 기지 미션 컴퓨터에 기록된 `mission_computer_main.log` 파일을 분석하여 사고의 원인을 파악한다.

해당 로그 파일에는 로켓 발사 준비부터 미션 종료까지의 주요 시스템 동작이 시간 순서대로 기록되어 있다.  
본 보고서는 로그 데이터를 기반으로 정상 동작 구간과 문제 발생 구간을 구분하고 사고의 원인을 분석한다.

---

## 2. 로그 분석 방법

Python 프로그램을 사용하여 `mission_computer_main.log` 파일을 읽고 전체 로그 내용을 확인하였다.

로그는 다음과 같은 형식으로 기록되어 있다.

timestamp,event,message

각 로그는 다음 세 가지 정보로 구성된다.

- **timestamp** : 이벤트 발생 시간
- **event** : 이벤트 수준 (INFO 등)
- **message** : 시스템 상태 메시지

로그는 시간 순서대로 기록되므로 시스템의 전체 동작 흐름을 파악할 수 있다.

---

## 3. 로그 흐름 분석

로그를 시간 순서대로 분석한 결과 로켓 미션은 다음과 같은 과정으로 진행되었다.

| 시간  | 이벤트              |
| ----- | ------------------- |
| 10:00 | 로켓 초기화 시작    |
| 10:30 | 로켓 발사 (Liftoff) |
| 10:57 | 지구 궤도 진입      |
| 11:05 | 위성 배치 성공      |
| 11:28 | 로켓 착륙 성공      |
| 11:30 | 미션 완료           |

위 로그 흐름을 통해 로켓 발사 및 위성 배치 과정은 정상적으로 수행되었으며 미션 자체는 성공적으로 종료된 것으로 확인된다.

---

## 4. 문제 발생 로그

미션 종료 이후 다음과 같은 이상 로그가 발견되었다.

첫 번째 로그는 산소 탱크의 상태가 불안정해졌음을 의미한다.  
이후 약 5분 뒤 산소 탱크 폭발이 발생한 것을 확인할 수 있다.

---

## 5. 사고 원인 분석

로그 흐름을 종합적으로 분석하면 사고는 다음 과정으로 발생한 것으로 판단된다.

1. 로켓 미션이 정상적으로 완료됨
2. 미션 종료 이후 산소 탱크 상태 이상 발생
3. 산소 탱크가 불안정 상태(Oxygen tank unstable)에 진입
4. 해당 문제를 해결하기 위한 시스템 대응 로그가 존재하지 않음
5. 약 5분 후 산소 탱크 폭발(Oxygen tank explosion) 발생

특히 산소 탱크 불안정 상태 이후 이를 해결하기 위한 시스템 로그가 발견되지 않은 점을 고려하면, 안전 제어 시스템 또는 자동 대응 시스템이 정상적으로 작동하지 않았을 가능성이 있다.

---

## 6. 결론

로그 분석 결과 로켓 발사와 미션 수행 자체는 정상적으로 완료된 것으로 확인되었다.

그러나 미션 종료 이후 산소 탱크가 불안정 상태에 진입하였으며, 해당 문제에 대한 시스템 대응이 이루어지지 않아 약 5분 뒤 산소 탱크 폭발 사고가 발생한 것으로 판단된다.

따라서 본 사고의 직접적인 원인은 **산소 탱크의 불안정 상태와 이에 대한 대응 실패**로 분석된다.
